# Security Policy

## Scope

This repository contains a reasoning skill and supporting documentation. It does not provide a network service, package binary, credential store, or runtime that should receive secrets.

## Sensitive Information

Do not include private keys, API keys, passwords, tokens, proprietary documents, customer data, medical records, financial account data, or other sensitive information in issues, pull requests, examples, evals, or discussions.

If an example requires realistic context, replace sensitive details with synthetic or anonymized data.

## Reporting Issues

For security-sensitive reports, avoid posting exploitable details publicly. Use GitHub's private vulnerability reporting or contact the repository owner through GitHub if that is available.

For ordinary documentation bugs, prompt-quality issues, unclear instructions, or eval failures, use GitHub Issues.

## AI Output Safety

This skill is designed to improve reasoning quality, but it does not make AI output automatically correct. High-stakes use cases should still require source verification, domain expert review, and explicit validation before action.
